Exercícios elaborados Capítulo 28

1. O que é a engenharia de software sala limpa? Engenharia de software sala limpa é uma abordagem que enfatiza a necessidade de agregar precisão ao software enquanto ele está sendo desenvolvido. A filosofia por trás da engenharia de software sala limpa é evitar a dependência de processos onerosos de remoção de efeitos, escrevendo incrementos de código corretos já na primeira vez e verificando sua precisão antes do teste. O modelo incorpora a certificação estatística de qualidade dos incrementos de código, à medida que vão se acumulando no sistema.



2. Como segue a abordagem sala limpa? À medida que cada incremento é certificado, ele é integrado no todo, por isso o sistema cresce com o tempo. Segue as tarefas: Planejamento de incremento, Coleta dos requisitos, Especificação box structure, Projeto formal, Verificação de correção, Geração de código, inspeção e verificação, Planejamento do teste estatístico, Teste de uso estatístico, Certificação.

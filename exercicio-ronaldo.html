1)Descreva a Engenharia de Software Sala Limpa. R: É uma abordagem que enfatiza a precisão ao software durante o desenvolvimento, visando incremento de códigos corretos já na primeira vez e verificando sua precisão antes do teste, sendo assim diminuindo a quantidade de ciclos de análise. Elevando a engenharia de software a outro nível, enfatizando a necessidade de provar a precisão. 2)Em relação a engenharia de software sala limpa explique a afirmação “O sistema cresce com o tempo”. R: A engenharia de software sala limpa usa uma versão especializada do modelo incremental de software, a medida que cada incremento é certificado, ele é integrado no todo, por isso a afirmação. 3)O que é a representação box Structure, utilizada na analise e modelos de projeto da abordagem sala limpa? R: Conciste em uma “caixa” que encapsula o sistema ou aspecto do sistema em um nível especifico de abstração, as caixas são divididas em 3 metodos, caixa preta a aqual tera uma analise externa, caixa de estado a qual encapsula dados de estado e operações e caixas clara que serão usadas para modelar o projeto mediante os dados da caixa de estado, finalizada está etapa sera iniciada a verificação de correção onde se passar será adicionado ao código final. 4)Descreva os Métodos Formais. R: Os métodos usam recursos descritivos da teoria dos conjuntos e notação lógica para possibilitar ao engenheiro de software criar uma definição clara dos fatos. 5)Compare métodos formais e engenharia sala limpa aos métodos de engenharia convencionais. R: Eles são mais difíceis de aprender doque os convencionais e representam um choque cultural para muitos profissionais, porém muitas vezes escutamos a pergunta, “porque não podemos escrever esse código certo de priemira”, essas técnicas poderiam ajudar com essa questão.
